# WhatsApp OpenCode CI/CD & Testing Strategy Plan

> **For Trae:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Establish a robust Git workflow, automated CI pipeline, and a manual integration testing guide to ensure quality before v2.0.0 release.

**Architecture:** 
- **Git Flow:** Feature branches -> PR -> Main (Stable). Tags trigger releases.
- **CI/CD:** GitHub Actions for automated testing on PRs and automated npm publishing on Tags.
- **E2E Testing:** Manual verification checklist for WhatsApp login (since automation is tricky without persistent sessions).

**Tech Stack:** GitHub Actions, npm scripts, Markdown (docs).

---

### Task 1: Document Contribution & Git Workflow

**Files:**
- Create: `CONTRIBUTING.md`

**Step 1: Define Workflow**
Create `CONTRIBUTING.md` detailing:
1.  Branch naming convention (`feature/`, `fix/`, `docs/`).
2.  Commit message convention (Conventional Commits).
3.  Pull Request process (Review required, CI must pass).
4.  Release process (Tagging triggers npm publish).

### Task 2: Setup GitHub Actions CI

**Files:**
- Create: `.github/workflows/ci.yml`

**Step 1: Create CI Workflow**
Define a workflow that runs on `push` to `main` and `pull_request` to `main`.
- Jobs:
  - `test`: Runs `npm ci`, `npm run lint`, `npm test`.
  - Matrix: Node.js 18.x, 20.x.

### Task 3: Setup GitHub Actions CD (Release)

**Files:**
- Create: `.github/workflows/release.yml`

**Step 2: Create CD Workflow**
Define a workflow that runs on `push` with tags `v*`.
- Jobs:
  - `build`: Runs tests.
  - `publish`: Publishes to npm (requires NPM_TOKEN secret).
  - `release`: Creates a GitHub Release with notes.

### Task 4: Create Manual Integration Test Guide

**Files:**
- Create: `docs/INTEGRATION_TEST_GUIDE.md`

**Step 1: Document Manual Steps**
Since we cannot automate scanning the QR code in CI easily, write a guide for the maintainer (you) to run locally before release:
1.  Clean install: `npm install -g .` (or `npm link`).
2.  Setup: `wao setup`.
3.  Start: `wao start`.
4.  Login: Scan QR code.
5.  Verify: Send "ping", send image, send voice.
6.  Teardown: `wao stop`, `wao status`.

### Task 5: Execute Local Integration Test (Interactive)

**Files:**
- None (Action)

**Step 1: Verify Locally**
Follow the guide created in Task 4 to verify the current build.
**Important:** This step requires user interaction (scanning QR code). We will guide the user to do this.
